# Cely

Cely is an ESL Patient intake tool that enables patients to communicate their most important concerns to providers, and helps save time for providers by synthesizing the highest priority concerns for their upcoming appointment. Made as an entry to Cerebral Valley Hackathon 7/18/2026.

It turns multilingual patient concerns into a clinician-ready, evidence-linked visit agenda grounded in Athena Preview, keeping the patient's words, chart evidence, uncertainty, and clinician approval connected from intake through handoff.

Cely is not a diagnostic chatbot. It preserves the patient's words, makes uncertainty visible, keeps patient-reported and chart-derived facts distinct, and requires human approval before any Athena write.

## Current demo

The current product slice is a multilingual pre-visit intake for new, returning, or long-absent outpatients. It includes:

- first-message language detection with manual confirmation when detection is uncertain;
- bounded clarification in the patient's language;
- separate, patient-confirmed visit concerns and an optional top concern;
- a same-language restatement and English clinician interpretation generated by Sonnet;
- Athena Preview context for appointments, problems, active medications, and allergies;
- an evidence-linked clinician handoff with visible rationale and citations;
- deterministic red-flag screening before the model for supported English, Spanish, and Tagalog phrases, plus a second screen over the English interpretation for other languages;
- clinician- or nurse-approved Preview note writeback behind a separate feature flag.

The detector recognizes many common written languages. Short or ambiguous messages retain the original text and ask the patient to confirm or enter a language. Localized intake controls are available for several widely used languages, with an English fallback for the rest.

Voice intake, SMS or email delivery, scanned-chart ingestion, ICD mapping, translated after-visit summaries, and persistent patient-data storage are not implemented in this demo.

### Data handling

Application state is session-scoped and held in memory; Cely does not add a persistent patient-data database. This is not a zero-retention claim. When the integrations are enabled, Anthropic processes submitted patient text and Athena Preview supplies chart context under those services' data policies. Resetting Cely clears local session state only.

## Three-minute demo

1. Choose `Tagalog shoulder + meds` and start Maya Santos's intake. Maya describes a shoulder that is `kumikirot`, morning dizziness, and stopping her blood-pressure medicine.
2. Cely detects Tagalog from Maya's first message, preserves her exact words, asks one bounded clarification, separates the concerns, and asks which one matters most to her.
3. Submit the confirmed intake. For the configured Maya scenario, Cely retrieves synthetic Athena Preview chart context and creates an evidence-linked clinician agenda. Maya's stated priority remains separate from clinical prioritization.
4. Approve the handoff. With the default configuration, Cely validates the payload and records a dry-run receipt without mutating Athena.
5. Run the red-flag replay, or enter a supported active red-flag phrase. Cely stops routine intake, displays emergency guidance, bypasses the model when the original language matches a deterministic rule, and performs no Athena write.

For a second routine path, choose `Tagalog foot pain`. The example preserves six months of left-foot pain with big-toe onset, an unspecified medication concern, a prior clinician's arthritis attribution, a patient-observed food association, and the patient's concern about kulam. The handoff does not diagnose gout, infer a medication, assert a dietary cause, or present the cultural or spiritual explanation as clinical fact.

## Run locally

Install dependencies and create a local configuration:

```bash
pnpm install
cp .env.example .env.local
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000). The default configuration uses deterministic fixtures and does not require external credentials.

Run linting, type checking, tests, and a production build:

```bash
pnpm verify
```

The test command uses Bun, so Bun must be installed to run the full verification suite.

## Live integrations

Athena and Anthropic can be enabled independently. Copy `.env.example` to `.env.local`, then set only the integration you want to use.

### Athena Preview

Cely uses Athena Preview's server-side two-legged OAuth flow:

- base URL: `https://api.preview.platform.athenahealth.com`;
- token endpoint: `POST /oauth2/v1/token`;
- scope: `athena/service/Athenanet.MDP.*`;
- reads: patient, appointments, problems, active medications, and allergies; and
- optional write: `POST /v1/{practiceid}/appointments/{appointmentid}/notes`.

Configure live reads with:

```bash
ATHENA_MODE=live
ATHENA_BASE_URL=https://api.preview.platform.athenahealth.com
NEXT_PUBLIC_ATHENA_CLIENT_ID_2_LEG=...
ATHENA_CLIENT_SECRET_2_LEG=...
ATHENA_PRACTICE_ID=1959870
ATHENA_DEPARTMENT_ID=150
ATHENA_DEMO_PATIENT_ID=946985
ATHENA_DEMO_APPOINTMENT_ID=2589077
```

Patient `946985` is the synthetic Maya Santos Preview record used by the golden path. Live chart retrieval is limited to the configured Maya scenario; other demo scenarios continue to use fixtures.

Keep `ATHENA_WRITEBACK_ENABLED=false` for a dry-run approval receipt. If writeback is enabled, the route accepts only clinician or nurse approval, verifies that the submitted appointment matches the configured patient and appointment, confirms that it remains booked and current, and writes only to Athena Preview.

The writeback route is a hackathon safeguard, not a production authorization system. Production use still requires authenticated users, a server-side approval record bound to the run, durable audit storage, and server-controlled note construction.

Cely refuses live Athena mode for any base URL other than Preview. The Athena secret and access token remain server-side, and upstream response bodies are not returned to the browser.

### Anthropic

Configure both AI stages with:

```bash
AGENT_MODE=live
ANTHROPIC_API_KEY=...
ANTHROPIC_INTERPRETATION_MODEL=claude-sonnet-5
ANTHROPIC_MODEL=claude-sonnet-5
```

`ANTHROPIC_INTERPRETATION_MODEL` generates the bounded bilingual review before patient confirmation. `ANTHROPIC_MODEL` generates the downstream clinician handoff. Both responses are schema-constrained. If live interpretation is unavailable, Cely stops at clarification rather than presenting source-language text as an English translation. A handoff generation failure falls back to an evidence-linked deterministic handoff for the same patient and run.

## Safety contract

- Use only synthetic or explicitly authorized Athena Preview data.
- Preserve original patient wording and keep it separate from translations, model output, and chart facts.
- Require patient confirmation of the native-language restatement before routine intake continues.
- Treat missing chart data as unavailable, never as a negative finding.
- Stop routine intake when a deterministic safety rule matches; never send that directly matched text to the model.
- Do not let the model diagnose, prescribe, approve itself, or write autonomously.
- Require every agenda recommendation to cite evidence supplied in the handoff.
- Require a clinician or nurse, explicit approval, Athena Preview, and the writeback feature flag for every live write.

## Project map

```text
src/app/                    Next.js UI and route handlers
src/components/             patient intake and clinician workflow UI
src/lib/ai/                 constrained interpretation and handoff generation
src/lib/athena/             Preview-only authentication and typed client
src/lib/demo/               deterministic demo fixtures
src/lib/workflow/           contracts, runner, evidence, metrics, and policy
tests/                      language, safety, policy, evidence, and workflow tests
```

The same workflow and policy gates can later support care-gap outreach, record migration, and post-visit follow-up without weakening human approval or evidence requirements.
